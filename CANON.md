# CANON — mounting a licensed knowledge base

The scaffold gives you the machine; a **canon** gives it depth: behavioral-science principles, pattern doctrine per surface (onboarding, forms, AI interfaces, enterprise…), design-system intake protocols, research methods, and validated fixtures. An instance runs honestly without one (the constitution alone must produce credible work — that's the degradation rule), but the canon is where leader-grade depth comes from.

## The mount contract

A canon drops into the repo root as sibling directories, and `AGENTS.md` routes into them:

```
principles/    — the behavioral "why", one canon-indexed file per family
patterns/      — proven tactics + anti-patterns per surface/use-case
frameworks/    — audit and generation methods (scans, scoring, lenses)
research/      — discovery + validation methods
design-system/ — craft doctrine: tokens, components, voice, intake protocols
ethics.md      — the line you never cross
```

## Upstream / downstream governance

- **Upstream (the canon maintainer):** ships doctrine updates — new patterns, refreshed standards (accessibility, platform), improved fixtures.
- **Downstream (your instance):** receives updates **propose-only**. Nothing upstream ever touches your voice layer (§B, §G, takes) — that separation is structural, enforced by `MAINTENANCE.md`'s two-channel rule.
- Your voice layer is yours. The canon license covers canon content only.
