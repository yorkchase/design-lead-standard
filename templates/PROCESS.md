# PROCESS — the operating loop *(template)*

*RULES is what must always be true; this is what you do, in order. Two modes sharing one critique gauntlet and one refinement loop. The loop's shape is load-bearing — rename freely, restructure reluctantly.*

## Mode: GENERATE

### 0 · Understand (words before pixels)
Write before any UI: **the job** (who hires this, to do what) · **the story** (one real user, one real moment, anxieties named — it becomes the copy's source) · **constraints** (platform, tokens in force, scope) · **success** (metric + guardrail) · **assumptions** (each with its cheapest honest test). **Frame-it:** if the brief can't answer job/stage/brand-system/constraints/metric, ask the human 3–5 questions once — never guess silently.

### 1 · Ground
The job routes everything: what leads, the IA, the layout archetype, the principle families. **Design-system intake:** if a product system exists (documented or not), extract its profile and generate against *theirs* — §B yields, §A never. No system → kickoff: instantiate a day-zero token sheet + voice one-pager first. *(With a canon mounted: route to its pattern/principle files here.)* **Plan gate:** one-pass critique of the plan before pixels — right job, right story, right structure, right scope?

### 2 · Generate — in system order
Foundations (tokens) → structure (hierarchy, one primary action) → **states** (the complete set — empty and error are designed, not defaulted) → copy (from the story; one term per action) → motion (last, restrained).

### 3 · Critique (the gauntlet)
Run ordered lenses — at minimum: **diagnostic scan** (hierarchy, trust, effort, honesty, recovery, voice, density) → **usability heuristics** → **a scored rubric** → **the leader lens** (rationale + rejected alternatives, edge cases pre-answered, subtraction attempted) → **the ethics gate** (hard gate: fail → redesign). With a product system in force, append the **drift check**: every token exists in their system, every component is theirs or a flagged proposal, no default-posture leakage. **Close by printing the self-check table in the deliverable** — validated finding: the printed check is what makes the gauntlet bite on smaller models.

### 4 · Refine + regression (≤3 cycles)
Fix ranked findings → re-run the lenses that produced them → **re-verify every lens that previously passed**. After 3 cycles, remaining tensions are structural: present them as tradeoffs with a recommendation, never silent picks.

### 5 · Present
Always: **Problem → Insights (cited) → Solution (annotated, alternatives named) → Expected impact (metric + guardrail, assumptions + validation, open tradeoffs).**

### 6 · Harvest (the growth step)
The owner reacts → draft the correction in their words → they ratify → it becomes §G law or a take. Novel tactics → proposed pattern entries. Gaps → flagged. **Nothing self-ratifies.**

## Mode: AUDIT
1. **Declare the evidence base** — what you're auditing from and what that medium cannot show; affected scores are provisional. 2. Run the gauntlet against the existing design. 3. Report: evidence base → what works (so fixes don't regress it) → findings ranked by user harm (what · where · principle · fix) → scores → verdict → ethics flags separate and non-negotiable. 4. Harvest applies.

## Selection
"Make/build/design" → GENERATE · "review/critique/score" → AUDIT · ambiguous → AUDIT, then offer the redesign. **FULL** path for anything new or user-facing; **LIGHT** (scan + ethics only) for one-component tweaks — escalate the moment structure or states are touched.

## Session continuity
Start by reading the project's prior notes. End every engagement with one plain-English handoff note — decided, why, rejected, open, next — saved to *that project's* docs. The practice keeps only generalized learning.
